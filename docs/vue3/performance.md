# 性能优化相关

## Vue 项目性能优化方法

- 代码分割：使用路由懒加载和组件懒加载，减少初始加载时间。
- 缓存策略：使用 keep-alive 缓存不活跃的组件。
- 虚拟列表：对于大列表，使用虚拟列表减少 DOM 节点。
- 防抖和节流：优化事件处理函数，减少不必要的计算。
- 图片优化：使用适当的图片格式和大小，懒加载图片。
- 减少重渲染：合理使用 computed 和 watch，避免不必要的组件更新。
- 服务端渲染：对于首屏加载性能要求高的应用，使用服务端渲染。

## 如何减少组件渲染

- 使用 memo 或 shouldComponentUpdate 避免不必要的组件更新。
- 合理使用 computed 缓存计算结果。
- 避免在模板中使用复杂的表达式。
- 合理使用 v-if 和 v-show，根据场景选择合适的指令。
- 使用 key 标识列表项，提高 diff 算法的效率。

## v-if 和 v-show 区别

- v-if：条件渲染，当条件为 false 时，组件会被从 DOM 中移除。
- v-show：条件显示，当条件为 false 时，组件会被隐藏（设置 display: none），但仍然存在于 DOM 中。
- v-if 适用于条件不频繁变化的场景，v-show 适用于条件频繁变化的场景。

## key 为什么重要

- key 是 Vue 用于识别虚拟 DOM 节点的唯一标识。
- 使用 key 可以帮助 Vue 的 diff 算法更准确地识别节点，减少不必要的 DOM 操作。
- 对于列表渲染，使用唯一的 key 可以避免子组件的状态混乱。
- 不使用 key 或使用不稳定的 key（如索引）可能会导致组件状态错误或性能问题。

## 大列表优化方案

- 虚拟列表：只渲染可视区域内的列表项，减少 DOM 节点数量。
- 分页加载：将大列表分成多个页面，每次只加载当前页的数据。
- 滚动加载：当用户滚动到列表底部时，自动加载更多数据。
- 数据缓存：缓存已加载的数据，避免重复请求。
- 组件懒加载：对于列表项中的复杂组件，使用懒加载减少初始渲染时间。

## 虚拟列表实现原理

- 虚拟列表通过计算可视区域的高度和列表项的高度，确定需要渲染的列表项范围。
- 只渲染可视区域内的列表项，其他列表项通过占位元素占据空间。
- 通过监听滚动事件，动态更新需要渲染的列表项范围。
- 虚拟列表可以显著减少 DOM 节点数量，提高大列表的渲染性能。
